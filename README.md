# 올인원 스케줄러 (Web + App) 설계안

## 목표
누구나 바로 이해할 수 있는 **시각 중심 일정관리 서비스**를 구축합니다.

- 4가지 뷰: 데일리 / 위클리 / 먼슬리 / 이얼리
- 시간순 타임라인 + 체크박스 목록 동시 지원
- 업무/사적 스케줄 분리
- 반복 일정, 메모, 태그, 통계 대시보드, 알림
- 5회 초과 사용 시 구독 전환
- 웹과 모바일 앱 실시간 동기화 (Supabase 기반)

## 권장 기술 스택
- **Frontend Web**: Next.js + TypeScript + Tailwind + shadcn/ui
- **Mobile App**: React Native (Expo) 또는 Flutter
- **Backend/BaaS**: Supabase (Auth, Postgres, Realtime, Storage, Edge Functions)
- **푸시 알림**: FCM(안드로이드), APNs(iOS), Expo Notifications
- **결제/구독**:
  - 앱: Google Play Billing
  - 웹: 토스페이먼츠 정기결제 + 웹훅

## 핵심 UX 원칙
1. **한눈에 이해**: 텍스트가 길지 않아도 시각적으로 상태 인지 가능
2. **가독성 보장**: 라이트/다크 모드 모두 명도 대비 충족
3. **빠른 입력**: 2~3탭 안에 일정 생성
4. **역할별 커스터마이징**: 프리랜서/학생/직장인별 카테고리 템플릿 제공

## 정보 구조 (IA)
- 홈
  - 오늘의 일정
  - 원형 진행률 위젯(완료율 색상 채움)
  - 업무/사적 토글
- 일정
  - 타임라인 뷰
  - 체크리스트 뷰
  - 데일리/위클리/먼슬리/이얼리 전환
- 대시보드
  - 이번 주 완료율
  - 카테고리별 시간
  - 반복 일정 이행률
- 노트
  - 일정별 메모/첨부
- 설정
  - 알림 On/Off
  - 구독 관리
  - 테마(라이트/다크/시스템)

## 권한 및 동기화
- 이메일/소셜 로그인 (Supabase Auth)
- 멀티디바이스 동기화 (Realtime subscription)
- 오프라인 캐시 후 온라인 복구 시 병합

## 구독 정책 (5회 무료 사용 후)
- 무료 사용자: 핵심 기능 체험 5회
- 6회부터 구독 팝업 노출
- 결제 성공 시 `subscription_status=active`
- 결제 실패/만료 시 읽기 전용 또는 제한 모드

## 사용자 맞춤 카테고리 예시
- 프리랜서: 유튜브, 인스타그램, 클라이언트A, 회계
- 학생: 국어, 수학, 영어, 과학, 자습
- 기본 제공 + 사용자가 자유롭게 추가/정렬/숨김 가능

## 알림 정책
- 작업별 시작 시간 알림
- 사전 알림(예: 10분 전) 옵션
- 반복 일정 알림 자동 생성
- 방해금지 시간대 설정
- 알림 개별 ON/OFF

## 원형 스케줄(시각화)
- 24시간 링 차트에 작업 블록 표시
- 완료 시 해당 블록 채도 상승
- 하루 완료율에 따라 중심 원 컬러 변경
- 스마트워치 확장을 고려해 단순 색상/아이콘 중심 설계

## 접근성/가독성 기준
- 본문 대비 4.5:1 이상, 큰 텍스트 3:1 이상
- 상태 색상만으로 의미 전달 금지 (아이콘/레이블 병행)
- 최소 터치 영역 44x44px
- 다크모드에서 회색 텍스트 과도 사용 금지
